Sobre el almacenamiento

Todos los datos viven en el localStorage del navegador donde estás ahora. Esto significa:
• Cada navegador / equipo tiene su propia base de datos.
• Los datos persisten incluso si cierras el navegador.
• Se borran si el usuario limpia su caché del navegador.
• No hay sincronización automática entre dispositivos.

Para sincronización en tiempo real entre toda tu organización (todos viendo los mismos proyectos y usuarios desde cualquier PC), hay que migrar a un backend gratuito como Firebase Authentication + Firestore o Supabase. Eso se puede hacer en una fase posterior sin tirar el portal actual — solo se conecta el sistema de auth al servicio externo.
